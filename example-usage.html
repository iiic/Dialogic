Dialogic v 0.1

Dialog element javascript working like native Notification

Just vanilla javascript, no jQuery or other libraries. Styles and examples included in repo. Same parameters as browser's default Notification() function

Rozdíly mezi Dialogic a Notification

  • Dialogic neobsahuje vlastnost permission, popup můžete zobrazit kdykoliv, bez nutnosti získávat oprávnění
  • Dialogic má ve výchozím nastavení (dá se změnit) option requireInteraction: true čili popup se automaticky neskrývají, na rozdíl od Notification, které mají ve výchozím nastavení requireInteraction: false (taktéž se dá změnit během zakládání instance Notification)
  • Dialogic zatím nemá implementované vlastnosti:
    • actions
    • badge

Inicializace

Dialogic:

Vložit do stránky, pro funkci scriptu Dialogic je jedno kam (tedy jestli do elementu head, na konec body, či kamkoliv)

<script type="module" src="dialogic.mjs?v=0.1" crossorigin="anonymous" data-integrity="sha256-todo"></script>
Notification:

Pro použití příkladů s API Notification jsou nutné oprávnění, bez nich se notifikace nezobrazí. Bez oprávnění se po volání Notification provede onerror event

<script defer>
	Notification.requestPermission();
</script>

V obou případech stejné chování. Pořadí je v obou případech od té nejnovější k té nejstarší. Tedy v zde uvedeném příkladu odspodu.

Využití tagu, k zobrazení oznámení/popup pouze 1x /// @todo : dokončit

Dialogic, dialog s tagem se nezobrazí znovu dokud se neobnoví stránka. V případě opakovaného volání se stejným tagem se nezobrazí 2. dialog, jen se u prvního změní obsah. Stejně jako v případě Notification je výchozí hodnota renotify nastaven na false, tedy nedojde k žádnému efektu zobrazejí, jen tichému a nenápadnému zaměnění obsahu za nový. 
<script>
	new Dialogic( 'dialog s tagem', {
		tag: 'some-id-text-or-number',
	} );
	setTimeout(function() {
		new Dialogic( 'jiný dialog se stejným tagem', {
			icon: 'img/crow.png',
			tag: 'some-id-text-or-number',
		} );
	}, "3000");
</script>
Notification, pokud má zpráva tag (a volbu renotify nastavneou na false, což je výchozí nastavení) a je volána vícekrát chování záleží na notifikačním centur vašeho OS, pokud z něj zprávu nesmažete, podruhé se nezobrazí, ale přesto druhé volání může změnit obsah notifikace (pokud je volána znovu s jiným textovým obsahem, ale stejným tagem): 
<script>
	new Notification( 'notifikace s tagem', {
		tag: 'some-id-text-or-number'
	} );
	setTimeout( function () {
		new Notification( 'jiná notifikace se stejným tagem', {
			icon: 'img/crow.png',
			tag: 'some-id-text-or-number',
		} );
	}, "3000" );
</script>
(opakované volání Notification vám zprávu znovu nezobrazí, doukd ji neodstraníte z notifikačního centra)

Použití tagu a renotify současně /// @todo : dokončit

V obou případech, zobrazím notifikaci/dialog a ten následně překryji dalším o 3s později. Rozdíl oproti předchozímu příkladu je v použití renotify: true které zajistí opakované zobrazní i s efektem.

Dialogic: 
<script>
	new Dialogic( 'dialog s tagem', {
		tag: 42,
	} );
	setTimeout(function() {
		new Dialogic( 'jiný dialog se stejným tagem', {
			icon: 'img/crow.png',
			tag: 42,
			renotify: true,
		} );
	}, "3000");
</script>
Notification Díky efektu by se mohlo zdát že jsou to 2 nezávislé notifikace po sobě, ale v notifikačním centru zjistíte že je tam pouze jedna. A po případném opakovaném stisku tlačítka ke spuštění kódu se napřed nestane nic (protože stejný tag a 1. notifikace nemá renotify) a po 3s se notifikace zobrazí (protože 2. notifikace v pořadí už renotify má): 
<script>
	new Notification( 'notifikace s tagem', {
		tag: 42,
	} );
	setTimeout( function () {
		new Notification( 'jiná notifikace se stejným tagem a renotify', {
			icon: 'img/crow.png',
			tag: 42,
			renotify: true,
		} );
	}, "3000" );
</script>

Automatické skrývání oznámení… možnosti

Dialogic, ve výchozím stavu popupy neskrývají (requireInteraction = true) v tomto příkladě je pro změnu ukázané, jak je možné popupy automaticky skrývat s (requireInteraction = false) po 6s (což je výchozí čas, dá se změnit pomocí globálního nastavení které je v příkladu níže): 
<script>
	new Dialogic( 'nadpis', {
		body: 'tento popup se sám zavře po 6s',
		requireInteraction: false
	} );
</script>
Notification, ve výchozím nastavení skrývá oznámení za 6s, tady v tomto kousku kódu nastavím, že nechci notifikaci skrývat nikdy: 
<script>
	new Notification( 'nadpis', {
		body: 'tato notifikace sama od sebe nikdy nezmizí, vyžaduje klik',
		requireInteraction: true
	} );
</script>

Navázané události

Dialogic: 
<script>
	let d = new Dialogic( 'nadpis', { body: 'na tento dialog jsou navázané události… alerty' } );
	d.addEventListener( 'click', () => { alert( 'event click' ); } );
	d.addEventListener( 'close', () => { alert( 'event close' ); } );
	d.addEventListener( 'error', () => { alert( 'event error' ); } );
	d.addEventListener( 'show', () => { alert( 'event show' ); } );
	// alternativně se dá zapsat jako (a fungovat to bude stejně):
	// d.onclick = () => { alert( 'event click' ); };
	// d.onclose = () => { alert( 'event close' ); };
	// d.onerror = () => { alert( 'event error' ); };
	// d.onshow = () => { alert( 'event show' ); };

Notification (důležité: často dochází k nepochopení kdy se vyvolá close event. Dialog s možnostmi ano / ne

Dialogic: 
<script type="module">
	let d = new Dialogic( 'otázka', {
		type: Dialogic.CONFIRM, // tohle zobrazí tlačítka ano / ne
		body: 'souhlasíte?'
	} );
	d.onclick = () => { alert( 'zvoleno ano' ); };
	d.onclose = () => { alert( 'zvoleno ne' ); };
</script>
Notification … nejde napodobit zcela přesně, ale na základě předchozího příkladu se dá udělat něco jako: 
<script>
	let n = new Notification( 'otázka', {
		body: 'souhlasíte?',
		requireInteraction: true
	} );
	n.onclick = () => { alert( 'zvoleno ano' ); };
	n.onclose = () => { alert( 'zvoleno ne' ); };
</script>

Timestamp… možnost přidání času v budoucnu i minulosti

V obou případech stejné chování. Tedy datum a čas se vypíše u daného objektu (dialogu / notifikace) v případě dialogu v levém spodním rohu, v případě notifikace v notifikačním centru. Tato vlastnost ale neslouží k odloženému zobrazení, vždy se jak notifikace tak dialog zobrazí okamžitě, nastavený čas je jen informací uživateli.

Dialogic: 
<script>
	const plus30min = Math.floor( new Date(Date.now() + 30 + 60 * 1000) );
	new Dialogic( 'nadpis', {
		body: 'Tento dialog se zobrazí okamžitě, vypíše čas o minutu později oproti aktuálnímu.',
		timestamp: plus30min
	} );
</script>
Notification: 
<script>
	const yesterday = Math.floor( new Date(Date.now() - 24 * 60 * 60 * 1000) );
	new Notification( 'včerejší', {
		body: 'tato notifikace předstírá včerejší datum',
		timestamp: yesterday
	} );
</script>
